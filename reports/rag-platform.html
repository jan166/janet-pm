01 · Product Vision

프로덕트 비전 · 배경

LLM의 가장 큰 한계는 "모델이 갖고 있지 않은 정보"입니다. 이걸 기업이 자기 데이터로 직접 메우는 파이프라인을 만드는 것이 RAG Platform의 출발점. 단순한 검색 + 생성을 넘어, 보안 · 데이터 연동 · 품질 평가까지 PaaS 한 세트로 묶는 것이 차별화 포인트입니다.

VISION 01

고객 데이터 보안 환경 우선

VPC 환경 · KMS 연동을 기본 전제로 설계. 금융 · 공공 · 대기업 등 보안 수준이 높은 B2B 고객이 신뢰할 수 있는 데이터 처리 경로를 제공.

VISION 02

논스톱 파이프라인

데이터 연동 → 전처리/OCR → 임베딩 → 검색 → LLM 연동 → 평가까지 하나의 Console에서 설정. "RAG 구축에 수 개월"이 아니라 "설정 몇 번에 PoC"를 목표로.

VISION 03

품질 관리 · Evaluation

Generation 품질은 운영 중에 변질되기 쉬운 영역. 정답셋 기반 정기 평가와 Metric 제공으로 "PaaS로 맡겨도 품질이 흔들리지 않는다"는 신뢰를 확보.

02 · Architecture

서비스 구성 · 파이프라인

Prompt Engineering

Query Rewriting 기획 · 프롬프트 실험

  • "Rewrite-Retrieve-Read" 프레임워크 논문 정독 후 기획자 관점으로 정리 — 기존 "Retrieve-Then-Read"의 한계(입력 텍스트와 필요 지식의 격차, 고정 쿼리, 복잡 질문 처리 어려움, 적응형 검색 부재)를 조직에 공유.
  • 프롬프트 품질 조건 5가지 정의 — 구체적 지시, 역할 부여, 예시 제공, 조건·제한, 요구사항 반복. 각 조건별 실제 프롬프트 예시와 함께 "AS-IS / 모두 강화 / 1~5 강화" 비교 실험 설계.
  • 테스트 방법론 · 지표 · 신뢰 수준 정의 태스크 도출 — 단순 프롬프트 변경이 아닌 "좋은 Query Rewriting이 나오게 하려면 어떤 테스트를 해야 하는가"의 방법론 체계화.
