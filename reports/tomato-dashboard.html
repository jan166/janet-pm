Tomato Dashboard

메일·캘린더·할 일·위키를 한 화면에 모으고, 뽀모도로 리듬으로 다시 정리한 개인 대시보드. 기본 NAVER Works UI가 묻어버리는 액션 아이템을 다시 표면으로 끌어올리기 위해 만들었습니다.

왜 만들었나

네이버 세일즈기획실에서 광고 스폰서십 패키지를 담당하면서, 하루에 처리해야 할 신호가 5개 surface(메일·캘린더·태스크·위키·메신저)에 흩어져 있었습니다. 기본 UI는 각자 잘 만들었지만, 한 사람의 하루 안에서 "지금 무엇을 해야 하는가"를 통합해서 보여주지 않았어요. 매일 5개 탭을 돌아다니다 보면 액션 아이템이 묻혔습니다.

무엇을 만들었나

  • 한 화면 통합: 메일 미회신 / 오늘 회의 / 마감 임박 태스크 / 최근 위키 변경을 한 페이지에 컴팩트하게 노출
  • 뽀모도로 리듬: 25분 집중 / 5분 휴식 사이클을 페이지 자체가 페이싱 (상단 타이머)
  • Mac cron + 로컬 Python: NAVER Works API를 15분 간격으로 폴링해 데이터 갱신
  • Telegram 알림 봇 연동: 우선순위 높은 신호는 텔레그램으로 푸시

운영 방식

완전히 로컬에서 동작합니다. ~/nworks-dashboard에 Python generate.py가 들어 있고, serve.sh가 정적 서버를 띄워줍니다. 사내 OSS GitHub(oss.navercorp.com/hyoju-cheong/work-dashboard-n)에도 미러링되어 있어 팀이 같은 패턴을 가져다 쓸 수 있게 만들어 두었습니다.

이후 확장

이 대시보드 위에서 자라난 작은 도구들이 몇 개 더 있습니다 — 월드컵 미회신 알림 시스템, Telegram 음성 → Claude 봇, Sales Trend Report 생성기. 각각 별도의 사이드 프로젝트로 정리될 예정.
