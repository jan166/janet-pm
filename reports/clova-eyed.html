01 · Product Vision

프로덕트 비전 · 배경

2020년 팬데믹으로 비대면 인증 시장이 급격히 확대되던 시점, 사내에서 개별 운영되던 Android · iOS SDK는 동일한 기능을 중복 개발하고 있었습니다. 이 둘을 하나의 일관된 UX · 동일한 엔진으로 통합하고, 상품화까지 이어나가는 것이 프로젝트의 출발점이었습니다.

VISION 01

비대면을 넘어선 가치

얼굴 인식 기반 전자 신원 확인 솔루션을 만들어, 비대면 시대 이후에도 계좌 인증 · 신분증 촬영의 프로세스 효율화 가치를 유지한다.

VISION 02

도메인 확장 가능성

금융뿐 아니라 여행 · 의료 · 중고거래 · 송금으로 버티컬을 확장해 수익을 다원화한다. 표준화된 SDK로 SMB 시장까지 내려가는 것이 목표.

VISION 03

Edge + Server Hybrid

요구되는 보안 수준에 따라 Edge(경량·빠른 추론)와 Server(고정확도)를 선택하거나 조합할 수 있는 유연한 아키텍처로 설계.

02 · Product Components

SDK를 구성하는 3가지 블록

FACE SDK는 얼굴 인식의 전 과정을, Card Detector SDK는 신분증·신용카드 캡처를 담당합니다. 이 둘을 시나리오 샘플 코드로 묶어 실제 구현 난이도를 낮췄습니다.

Fx

FACE SDK

얼굴 검출 · 키포인트 · 인식 · 안티스푸핑 파이프라인

  • FRVT(미국) · KISA(한국) 인증을 획득한 모델
  • Edge (CPU only) ~ Server (T4 GPU) 선택 가능
  • 5-keypoint · 106-keypoint 검출 지원
  • Face motion · 스티커 · 메이크업 구현 기반
  • Face occlusion(마스크·코·입·눈) 검출
Cd

Card Detector SDK

신분증 · 신용카드 자동 촬영 및 품질 판정

  • 신용카드 국제 규격 1.5858:1 사각형 자동 검출
  • 블러 · 기울기 · Glare · Shaky 품질 판정
  • 측면 인식 지원 (75° – 110°)
  • OpenCV(AOS) · Vision Framework(iOS) 기반
  • 여권 MRZ 영역 인식 beta
SC

Scenario Samples

eKYC 구현을 위한 실전 샘플 코드

  • FACE Compare: 신분증 얼굴 ↔ 실시간 셀카 (threshold 조정 가능)
  • Credit Card OCR: 블러·기울기 자동 보정 후 인식
  • eKYC 전체 플로우: OCR → Compare → Liveness
  • NAVER Pay · LINE Pay 실제 시나리오 기반

LEARNED

계약 관리에서의 교훈 — "우리에게 중요한 조항은 먼저 제안하는 게 낫다"는 판단. 구매팀은 기술을 100% 이해하기 어려우므로 기획자가 계약서 갱신 때마다 정독하고 필수 조항(예: 2% 이상 검수 오류 시 전수 반납)을 직접 제안하는 것이 리스크를 줄이는 가장 빠른 길이라는 것을 이 프로젝트에서 체화.

데이터 구축에서의 교훈 — 단순 수량 관리가 아니라 "이 데이터가 어떤 모델의 어떤 문제를 풀기 위한 것인지"를 작업자까지 이해하게 만드는 것이 결과물 품질의 핵심. KPI는 작업자가 지치지 않는 수준에서 시작해 숙달에 따라 단계적으로 올려야 한다.
